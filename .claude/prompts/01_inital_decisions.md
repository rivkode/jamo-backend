## Step 2: 핵심 의사결정 (Architecture Decisions)

Step 1에서 파악한 기존 세팅을 전제로, 다음 두 가지 핵심 의사결정을 진행합니다.
각 의사결정은 **옵션 제시 → 트레이드오프 분석 → 사용자에게 결정 요청 → ADR 기록** 순서로 진행합니다.

### 의사결정 #1: 인증 아키텍처

**다뤄야 할 질문**
1. 인증을 **별도 서비스(Auth Service)** 로 분리할 것인가, 아니면 각 서비스가 토큰 검증만 수행하는 형태로 갈 것인가?
2. 별도 분리한다면 **Spring Cloud(예: Spring Cloud Gateway + Spring Authorization Server)** 를 도입할 것인가, 아니면 더 가벼운 구성(자체 JWT 발급 서비스 + 게이트웨이 없이 각 서비스 직접 노출)으로 갈 것인가?
3. 토큰 종류: **JWT(stateless)** vs **Opaque Token + Redis 세션** — 어느 쪽인가?
4. 사용자(`user`) 도메인과 인증(`auth`) 도메인의 경계: 분리할 것인가, 통합할 것인가?

**제시해야 할 옵션 (최소 3가지)**
- **Option A**: 별도 Auth Service + Spring Cloud Gateway + Spring Authorization Server (full-stack)
- **Option B**: 별도 Auth Service (가벼운 자체 구현, JWT 발급) + 게이트웨이 없이 각 서비스가 JWT 직접 검증
- **Option C**: Auth를 별도 서비스로 두지 않고, User 서비스가 인증 책임도 함께 담당

각 옵션에 대해 다음을 분석해주세요:
- 장점 / 단점
- 30개 API + 5개 도메인 규모에서의 적합성
- 운영 복잡도 (인프라, 모니터링, 장애 격리)
- 향후 확장성 (소셜 로그인 추가, 외부 API 노출 등)
- Spring Cloud 도입 시 학습 곡선과 의존성 부담

**중요**: 단정하지 말고 **사용자가 답해야 할 질문 목록**을 함께 제시하세요.
예: "외부 시스템에 API를 공개할 계획이 있는가?", "MFA/소셜 로그인 요구사항이 있는가?"

---

### 의사결정 #2: 서비스 분할 기준

**다뤄야 할 질문**
1. 현재 도메인(`auth`, `diary`, `sentence`, `chat`, `user`)을 **각각 독립 서비스**로 둘 것인가, 아니면 일부를 **묶을 것**인가?
2. 묶는다면 **어떤 기준**으로 묶는가? (Bounded Context 응집도, 데이터 공유 빈도, 트랜잭션 경계, 팀 규모, 배포 주기)
3. **30개 API 규모에서 적정 서비스 개수는 몇 개**인가? (과분할 경계 주의 — 이전 대화에서 합의됨)

**도메인별 사전 분석을 먼저 제시해주세요**
각 도메인(`auth`, `diary`, `sentence`, `chat`, `user`)에 대해:
- 추정되는 책임과 주요 엔티티
- 다른 도메인과의 상호작용 (예: `diary`가 `sentence`를 포함하는지, `chat`이 `user`를 참조하는지)
- 독립 배포 필요성 (트래픽 패턴, 변경 빈도)

**그 다음 분할 옵션 제시 (최소 3가지)**
- **Option A**: 5개 도메인 = 5개 서비스 (1:1 매핑, 가장 세분화)
- **Option B**: 응집도 기반 통합 — 예: `diary + sentence`를 하나로, `auth + user`를 하나로, `chat` 별도 → 3개 서비스
- **Option C**: 시작은 모놀리식 모듈러(Spring Modulith) → 운영 후 분리 → 1개 배포 단위, 모듈 경계만 강제

각 옵션에 대해:
- 운영 복잡도 (배포 파이프라인, 모니터링, 분산 트랜잭션)
- DDD 관점의 Bounded Context 정합성
- 30개 API에 대한 인프라 비용 (5개 서비스 = DB 5개? 인스턴스 5개?)
- 향후 도메인 추가 시 확장성
- 데이터 일관성 처리 (Saga, Outbox 필요성)

**사용자가 답해야 할 질문 목록을 함께 제시하세요**
예: "diary와 sentence의 관계는? sentence가 diary에 종속적인가, 독립 도메인인가?", "팀 규모는?", "각 도메인의 트래픽 차이가 큰가?"
